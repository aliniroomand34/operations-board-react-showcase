Use these as **honest Mid+ / Senior-ready** lines. Do not inflate into “expert
TypeScript,” “senior testing expert,” or “production-grade responsive design.”

## Primary project bullets (pick 2–3)

- Built an AI-assisted React operations dashboard showcase from a real private
  SaaS codebase, anonymizing domain language while preserving modular architecture
  (UI / Logic / API / helpers).
- Designed a mock API boundary and synthetic dataset so the board demo runs as a
  static site with loading, error, and empty states — no secrets or live backends.
- Migrated the Operations Board slice to TypeScript (strict domain types, helpers,
  hook, and UI props) as part of a Senior-ready learning track, not as an expert claim.
- Added Vitest unit tests for domain helpers and mock API transitions, plus React
  Testing Library coverage for board workflows (assign, complete, recovery states).
- Documented architecture decisions (ADRs), CI (typecheck / lint / test / build),
  and desktop-focused demo scope so recruiters can evaluate craft without keyword stuffing.
